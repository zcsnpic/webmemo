## 1. 产品概述

### 1.1 产品定位

**回忆录客户端**是一款基于 Web 的单页应用（SPA），专为长辈打造轻量化、低门槛的记忆素材采集工具。核心定位为**回忆录前期素材收集终端**，仅负责通过AI辅助对话挖掘记忆碎片、留存原始口述内容，同步完成素材的存储与标记，所有素材统一归档留存，用于后期整合写作与AI辅助生成完整图文回忆录。

### 1.2 目标用户

| 属性 | 描述 |
|------|------|
| 年龄 | 70岁左右 |
| 操作能力 | 有比较强的打字和口述能力 |
| 使用方式 | 独立使用，无需子女辅助 |
| 视觉需求 | 界面字体较大，层次分明，减少视觉负担 |

### 1.3 核心功能

- **对话式素材采集**：摒弃纯文本输入模式，采用对话式交互，降低长辈创作门槛，轻松留存零散记忆。

- **AI辅助记忆挖掘**：三档可调AI参与度（L1记录档/L2倾听档/L3互动档），从被动记录到主动引导，深度挖掘被遗忘的记忆细节。

- **素材编辑整理**：提供独立的编辑区，支持标题编辑、内容撰写、保存等功能。

- **个性化素材整理**：支持自定义AI文风提示词，初步梳理口述素材，保留原始情感与细节。

- **精准记忆标注**：构建专属人名/地名名词库，人名支持录入曾用名、别名、常用称谓，让AI精准识别不同称谓指向同一主体。

- **实时名词校准**：每轮采访对话结束后，自动识别模糊专有名词，一键确认补充信息并同步入库。

### 1.4 技术栈

- **前端**：HTML5 + CSS3 + JavaScript（模块化结构）
- **存储**：本地存储 + GitHub Repository（纯文件存储）
- **AI 接口**：Deepseek API（默认）
- **文件格式**：Markdown (MD) + JSON
- **GitHub认证**：Personal Access Token（PAT）+ localStorage

### 1.5 用户角色

本产品为**单用户模式**，用户可进行所有操作。

| 角色 | 描述 | 权限 |
|------|------|------|
| 用户 | 老人（口述者） | 全部功能 |

5. **名词库**：统一管理全局名词库，支持全项目复用

## 4. 核心功能模块

### 4.1 栏目与素材管理

#### 4.1.1 栏目管理

- **创建栏目**：点击"新建栏目"按钮，输入栏目名称
- **编辑栏目**：悬停显示编辑/删除按钮
- **折叠/展开**：点击栏目头部切换展开状态
- **栏目显示**：显示栏目名称和素材数量

#### 4.1.2 素材管理

- **创建素材**：点击"新建"按钮，选择栏目后创建
- **选择素材**：点击素材项选中并加载
- **编辑素材**：在编辑区进行编辑
- **删除素材**：悬停显示删除按钮

### 4.2 AI 交互模式（三档可调）

#### 4.2.1 档位定义

| 档位 | 名称 | 适用场景 | AI响应策略 |
|------|------|----------|------------|
| L1 | 记录档 | 长辈口述不想被打断 | 仅回复"收到"/"嗯"，不做任何追问 |
| L2 | 倾听档 | 情感陪伴、情绪安抚 | 简短共情回应，不提问题 |
| L3 | 互动档 | 深度挖掘记忆细节 | 主动引导+生成3个追问问题 |

#### 4.2.2 各档位响应

```
L1 记录档:
├── 回复字数: 5字以内
├── 回复内容: "收到"/"嗯"/"好的，请继续"
├── 追问行为: 无
└── 名词处理: 仅标记，不弹窗

L2 倾听档:
├── 回复字数: 10-30字
├── 回复内容: "听起来很温馨"/"那确实很难忘"
├── 追问行为: 无
└── 名词处理: 标记+弹窗确认

L3 互动档:
├── 回复字数: 50-150字
├── 回复内容: 深度回应 + 3个引导性问题
├── 追问行为: 用户可点击问题继续深挖
└── 名词处理: 标记+弹窗确认+一键入库
```

### 4.3 采访区功能

#### 4.3.1 消息展示

- **AI消息**：左侧显示，带头像，底部可选追问问题
- **用户消息**：右侧显示
- **动画效果**：新消息淡入动画

#### 4.3.2 追问功能（L3档位）

对话结束后，AI生成3个预设追问问题，用户点击后自动发送追问。

### 4.4 编辑区功能

#### 4.4.1 标题编辑

- 输入框显示当前标题
- 失焦自动保存

#### 4.4.2 内容编辑

- 大面积文本编辑区
- 支持多行输入

#### 4.4.3 保存功能

- 点击保存按钮
- 显示保存状态（保存中/已保存）

#### 4.4.4 AI悬浮助手

- **位置**：编辑区右下角固定
- **显示**：始终显示，无需点击展开
- **功能**：输入指令调整文章内容
- **范围**：仅编辑区显示，采访区不显示

### 4.5 右侧栏 - 稿件属性

#### 4.5.1 时间信息

- **自然语言输入**：用户输入时间描述（1985年夏天、春节前一周等）
- **AI分析按钮**：点击后分析输入的时间描述
- **属性展示**：显示解析后的date和period标签

#### 4.5.2 标签管理

- **标签展示**：以标签形式展示已有标签
- **添加标签**：输入框输入后回车添加
- **移除标签**：点击标签上的×移除
- **限制**：最多10个标签

#### 4.5.3 本文名词库

- **折叠列表**：点击展开/收起详情
- **信息展示**：名称、别名、称谓
- **操作按钮**：编辑、删除

### 4.6 名词库管理

#### 4.6.1 全局名词库页面

- **触发**：点击左侧栏底部"名词库"入口
- **页面结构**：左侧分类导航 + 右侧词条列表
- **分类支持**：全部、人物、地点、物品
- **搜索功能**：支持按名称搜索

#### 4.6.2 词条详情弹窗

- **查看模式**：查看词条详细信息
- **编辑模式**：编辑词条内容
- **保存/取消**：确认或放弃修改

## 9. 本地保存规范

### 9.1 GitHub仓库结构

```
/
├── articles/              # 素材文档目录
│   ├── {栏目名}/
│   │   ├── {素材id}.md   # 素材文件（YAML头+正文）
│   │   └── ...
├── conversations/        # AI对话历史目录
│   └── {素材id}.json     # 对话历史文件
├── glossary.json          # 全局名词库
└── index.json            # 素材索引文件
```

### 9.2 文件命名规则

编辑区生成的Markdown文档采用以下命名规则：

```
{栏目名称}/{素材标题}_{日期}.md
```

**规则说明：**
- 栏目名称：素材所属的栏目，中文转拼音或保留中文
- 素材标题：用户输入的素材标题，进行URL安全处理
- 日期：素材创建或修改日期，格式为 ISO 8601 (YYYY-MM-DD)

**示例：**
```
童年/夏夜乘凉_1985-07-15.md
家庭/父亲的故事_1970-01-01.md
```

### 9.2 存储路径要求

```
/
├── articles/              # 素材文档根目录
│   ├── 童年/
│   │   ├── 夏夜乘凉_1985-07-15.md
│   │   └── 过年_1988-02-15.md
│   ├── 家庭/
│   │   └── 父亲的故事_19700101.md
│   └── ...
├── images/               # 图片素材目录（后续迭代）
├── glossary.json          # 全局名词库
└── config.json           # 系统配置文件
```

**路径规范：**
- 所有素材存储在 `articles/` 目录下
- 按栏目名称创建子目录
- 子目录名称与栏目名称保持一致
- 不允许在根目录直接存放素材文件

### 9.3 文件格式标准

素材文档采用标准Markdown格式，包含YAML头部元数据。

**文件结构：**
```
---
[元数据区域]
---

[正文内容区域]
```

### 9.4 元数据格式

YAML头部必须包含以下字段：

```yaml
---
title: 夏夜的星空与恐惧        # 素材标题（必填）
original_title: 夏夜乘凉       # 用户原始输入标题（可选）
date: "1985-07-15"           # 记忆日期（必填），ISO 8601格式
period: 1985年夏季            # 时间段标签（可选）
location: 安徽砀山            # 记忆发生地点（可选）
tags:                         # 素材标签（可选）
  - 童年
  - 邻里
  - 夏夜
characters:                   # 关联人物列表（可选）
  - 王刚
  - 李氏
characters_detail:            # 人物详细信息（可选）
  - name: 王刚
    aliases: 王建军
    addresses: 王科长, 老王
    relation: 邻居
  - name: 李氏
    aliases: 李桂花
    addresses: 奶奶
    relation: 祖母
summary: 回忆儿时夏夜邻里乘凉，因听鬼故事不敢去公厕的往事。
ai_mode: L3                   # AI对话档位（必填）
dialogue_turns: 8             # 对话轮次（必填）
created_at: 2024-01-15T08:30:00Z  # 创建时间（必填），UTC
updated_at: 2024-01-15T10:45:30Z  # 更新时间（必填），UTC
---
```

### 9.5 正文内容格式

正文内容为标准Markdown格式：

```markdown
# 夏夜的星空与恐惧

那时候我还在上幼儿园，每到夏天的晚上，妈妈就会在院子里摆张小桌子，
全家人围坐在一起乘凉。院子里种着一棵大槐树，树叶在晚风中沙沙作响。

邻居们也都喜欢出来乘凉，王刚一家就住在我家隔壁...

（正文内容继续...）
```

**正文格式要求：**
- 使用标准Markdown语法
- 标题使用 `#` 标记
- 段落之间留有空行
- 不使用复杂的HTML嵌入

### 9.6 自动保存机制

**保存触发条件：**
- 用户点击"保存"按钮
- 内容编辑后超过5秒无操作（自动保存）
- 切换栏目或素材时

**保存状态反馈：**
- 保存中：显示"保存中..."状态
- 已保存：显示"已保存"状态，持续2秒后恢复默认
- 保存失败：显示错误提示

## 10. GitHub上传逻辑

### 10.1 权限验证机制

#### 10.1.1 GitHub PAT授权

用户在设置中输入GitHub Personal Access Token：

```
授权流程：
1. 用户在设置页面输入GitHub PAT
2. 应用验证Token有效性
3. Token存储于浏览器localStorage
4. 使用Token访问GitHub API
```

#### 10.1.2 权限范围

应用申请的GitHub权限：

| 权限 | 用途 |
|------|------|
| repo | 读写私有仓库 |
| read:user | 读取用户信息 |

#### 10.1.3 GitHub仓库归属

- **同一账号使用**：所有同步基于同一GitHub账号
- 不支持多账号协作
- 单用户模式，无需冲突合并

### 10.2 分支管理策略

#### 10.2.1 分支结构

```
main分支：稳定版本，所有素材的最终来源
  └── 存储所有已发布的素材文件

develop分支：开发中的素材同步
  └── 用于暂存未确认的更改（可选）
```

#### 10.2.2 分支策略

- **默认使用 `main` 分支**
- 不使用Pull Request流程，直接提交
- 单用户模式，无需冲突合并

### 10.3 文件提交规范

#### 10.3.1 提交信息格式

采用语义化的提交信息格式：

```
{操作类型}: {简短描述}

[可选的详细说明]
```

**操作类型：**
| 类型 | 说明 |
|------|------|
| feat | 新增素材 |
| update | 更新素材内容 |
| delete | 删除素材 |
| glossary | 名词库更新 |

**示例：**
```
feat: 新增素材「夏夜乘凉」

- 栏目：童年
- 日期：1985年
- 标签：童年、邻里、夏夜
```

### 10.5 GitHub API调用

#### 10.5.1 获取文件内容

```javascript
// 检查文件是否存在
GET /repos/{owner}/{repo}/contents/{path}
Authorization: Bearer {token}
```

#### 10.5.2 创建或更新文件

```javascript
// 创建新文件
PUT /repos/{owner}/{repo}/contents/{path}
Authorization: Bearer {token}

Body:
{
  "message": "feat: 新增素材「夏夜乘凉」",
  "content": "{base64编码的文件内容}",
  "sha": "{file_sha}"  // 更新时需要提供，创建时不需要
}
```

#### 10.5.3 响应处理

**成功响应 (200/201):**
```json
{
  "content": {...},
  "commit": {
    "sha": "abc123...",
    "message": "feat: 新增素材「夏夜乘凉」"
  }
}
```

**失败响应 (401/403/404):**
- 401：Token无效或过期，需要重新授权
- 403：权限不足
- 404：仓库或路径不存在
