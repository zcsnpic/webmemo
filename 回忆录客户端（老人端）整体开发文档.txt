# 回忆录客户端（老人端）整体开发文档

## 1. 产品概述

### 1.1 产品定位

**回忆录客户端（老人端）**是一款基于 Web 的单页应用（SPA），专为长辈打造轻量化、低门槛的记忆素材采集工具，核心定位为**回忆录前期素材收集终端**，仅负责通过AI辅助对话挖掘记忆碎片、留存原始口述内容，同步完成图片素材的存储与标记，所有素材统一归档留存，用于后期整合写作与AI辅助生成完整图文回忆录，不承担最终文稿编辑、排版、插图等深加工功能，最终构建结构化、可追溯、可长期保存的个人数字记忆素材库。

### 1.2 核心价值

- **零门槛素材采集**：摒弃纯文本输入模式，采用对话式交互，降低长辈创作门槛，轻松留存零散记忆。

- **AI辅助记忆挖掘**：三档可调AI参与度，从被动记录到主动引导，深度挖掘被遗忘的记忆细节，丰富素材维度。

- **永久安全素材存储**：所有文本素材、图片素材、配置文件均存储于自有GitHub仓库，依托Git实现版本控制与异地备份，素材永不丢失。

- **个性化素材整理**：支持自定义AI文风提示词，初步梳理口述素材，保留原始情感与细节，适配后期写作需求。

- **精准记忆标注**：构建专属人名/地名名词库，人名支持录入曾用名、别名、常用称谓，让AI精准识别不同称谓指向同一主体，避免记忆素材出现人物混淆，保障素材真实性。

- **实时名词校准**：每轮采访对话结束后，自动识别模糊专有名词，一键确认补充信息并同步入库，杜绝记忆信息偏差，提升素材精准度。

### 1.3 技术栈

- **前端**：HTML5 + CSS3 + JavaScript（Vue/React 任选）

- **存储**：GitHub Repository（纯文件存储，无数据库）

- **AI 接口**：对接大模型 API（用于记忆引导、对话回应、素材初步梳理、语义理解、模糊名词识别）

- **文件格式**：Markdown (MD) + JSON

## 2. 整体架构

### 2.1 架构图（文字描述）

```Plain Text
用户浏览器 <--> Web 前端应用 <--> GitHub API (素材读写与备份)
                                      ^
                                      |
                                      v
                              大模型 API (记忆引导、素材梳理、模糊名词识别)
                                      ^
                                      |
                                      v
                              专有名词库 (实时同步、一键入库、智能匹配)
```

### 2.2 数据流向

1. **用户操作**：在前端进行AI对话记录、素材编辑、图片上传与标记。

2. **实时保存**：用户输入/修改后，自动提交至GitHub，无需手动点击保存，杜绝素材丢失。

3. **AI 调用**：前端将采访区原始对话内容发送至大模型，获取记忆引导、素材初步梳理结果，同步识别模糊专有名词。

4. **名词确认与入库**：用户确认模糊名词信息，一键同步至专有名词库，更新词库数据。

5. **素材读取**：前端通过GitHub API拉取仓库素材，渲染素材列表与原始内容，供后续查看与导出。

## 3. 页面布局（三栏式）

### 3.1 左侧栏：素材列表区

- **功能**：展示所有记忆素材文档，支持分类管理，快速定位目标素材。

- **结构**：
        

    - 顶部：新建素材按钮、搜索框。

    - 主体：自定义一级目录分组（如：童年、家庭、工作），按记忆场景归类素材。

    - 列表项：点击加载对应MD素材文件至中间内容编辑区。

- **特性**：
        

    - 分组仅为UI归类，底层为文件系统目录，与仓库存储结构一致。

    - 支持拖拽排序、重命名分组，方便素材规整管理。

### 3.2 中间栏：素材编辑区（核心）

分为上下两个可编辑区域，支持实时切换与联动，仅负责原始素材记录与初步梳理，不做最终文稿深加工；新增**名词确认弹窗**，每轮对话结束后自动触发，完成模糊信息校准。

#### 3.2.1 采访区（原始对话素材）

- **用途**：记录用户与AI的原始对话内容，是回忆录的核心素材库，完整保留口述原貌。

- **特性**：
        

    - **实时自动保存**：用户输入/修改后，自动提交至GitHub，无需手动点击保存，杜绝素材丢失。

    - **AI 交互面板**：顶部切换AI参与度（记录/倾听/互动），适配不同记忆记录场景。

    - **编辑功能**：基础富文本编辑（支持加粗、换行），仅用于素材内容修正，不做排版美化。

    - **模糊名词提醒**：对话过程中实时标记模糊专有名词，对话结束后集中弹窗确认，不打断回忆流程。

#### 3.2.2 素材梳理区（初步整理文稿）

- **用途**：AI基于原始采访素材，初步梳理后的规整文本，仅作为后期写作的基础底稿，不生成最终回忆录。

- **特性**：
        

    - **手动微调**：支持用户手动修改、修正梳理后的文本，保留核心记忆信息。

    - **默认提示词**：顶部显示当前生效的AI素材梳理指令（可在设置中修改），统一梳理风格。

    - **指令微调框**：底部内置输入框，用户可发送精简、补充细节等指令，AI原地优化素材内容。

    - **素材元数据头部**：文档最前部自动生成标题、时间标签、摘要，标注素材核心信息，方便后期检索。

    - **名词关联展示**：梳理文稿中，已入库专有名词自动高亮， hover可查看完整词库信息，便于核对修正。

### 3.3 右侧栏（可选 / 抽屉式）

- **功能**：素材辅助工具面板，默认收起，需要时展开，仅服务于素材管理与标注。

- **模块**：
        

    - **图片素材库**：浏览、上传、标记所有记忆相关图片，仅完成图片存储与信息标注，支持关联人物全称+别名/曾用名标注，不做插图操作。

    - **名词库**：查看、添加、编辑各类专有名词，人名板块支持完善标准姓名、曾用名、别名、常用称谓，支持模糊搜索快速定位人物信息，保障素材标注精准度；同步展示近期新增、确认的名词，方便快速修改。

    - **素材标签**：编辑当前素材的时间、性质、关联人物（含别名/曾用名）等标签，便于后期素材筛选与整合。

## 4. 核心功能模块

### 4.1 AI 交互模式（三档可调）

1. **记录档（Passive）**

    - AI仅作为被动记录员，用户输入口述内容后仅回复“收到”，不做额外回应。

    - 无任何主动引导，纯粹完整记录用户口述内容，保留原始素材。

    - 对话结束后自动触发模糊名词识别，弹窗完成信息确认，不干预记录流程。

2. **倾听档（Attentive）**

    - AI给予简短、共情的回应（如：“听起来很温馨”、“确实很吓人”），安抚情绪，助力记忆表达。

    - 不主动提问，仅提供情绪价值，适配用户平缓回忆的场景。

    - 同步识别对话中模糊名词，结束后统一确认，兼顾情绪安抚与信息精准度。

3. **互动档（Proactive）【推荐】**

    - AI深度回应内容，并在每次对话结束后，**生成3个引导性问题**，挖掘记忆细节。

    - 用户点击问题继续深挖记忆，或选择“不回答”跳过，操作简单便捷。

    - 核心目的：帮助用户回忆起更多被遗忘的细节，丰富记忆素材维度。

    - 引导提问结束后，自动识别本轮对话模糊信息，快速完成名词确认与入库，联动后续记忆挖掘。

### 4.2 智能素材生成

- **新建素材流程**：
        

    1. 用户点击“新建”，输入**素材简要标题**（如：夏夜乘凉），标注记忆主题。

    2. 进入采访区，与AI开启对话，记录原始口述记忆。

    3. 单轮对话结束，AI自动识别模糊人名、地名、称谓、旧称等信息，弹出**名词确认弹窗**。

    4. 用户补充完整信息，点击**一键加入词库**，信息自动同步至专有名词库，无需手动跳转编辑。

    5. 内容记录完成后，点击“梳理素材”。

    6. AI自动分析原始对话内容，结合已校准名词库，生成基础素材信息：
                

        - **优化素材标题**

        - **时间/时段标签**（如：幼儿园时期）

        - **素材核心摘要**

        - **初步梳理文稿**（内置校准后专有名词，无信息偏差）

    7. 以上元数据与梳理文稿自动置于素材梳理区头部，完整归档单条记忆素材。

### 4.3 图片素材管理系统

#### 4.3.1 图片上传与标记（核心功能）

- 支持本地图片批量上传至GitHub `/images` 目录，完成图片素材永久存储。

- 上传时/上传后，可为图片添加完整元数据标记，关联对应记忆素材：
        

    - 文件名、图片描述、拍摄/记忆时间、地点、关联人物、自定义记忆标签。

    - 标记时可直接调用已确认入库的专有名词，无需重复录入。

- 所有图片元数据单独归档，同时嵌入对应记忆素材MD文档的YAML头信息中，实现图文素材绑定，用于后期AI辅助整合生成完整图文回忆录。

### 4.4 专有名词库（Glossary）

- **数据结构**：存储为 `glossary.json` 文件，按人名、地名、物品名、关系称谓分类存储，其中人名模块新增曾用名、别名、称谓字段，适配长辈社交、工作场景中的多称谓习惯。

- **核心内容**：
        

    - 基础分类：人名、地名、物品名、亲属/社交关系称谓

    - 人名专属字段：标准姓名、曾用名、别名、常用称谓、备注（如身份、关联关系），例：标准姓名王刚，曾用名王建军，别名/工作称谓王科长

    - 其他分类：地名含曾用地名、别称，物品名含俗称、旧称

- **运行机制**：
        

    - 用户可手动录入：支持批量添加、单个编辑，填写标准姓名的同时，补充曾用名、别名、日常称谓等信息，适配长辈记忆习惯

    - AI智能识别补全：AI在对话、素材梳理时，自动识别文本中出现的别名、称谓、曾用名、模糊表述，关联对应标准姓名，未入库名词实时提示用户补充完整信息

    - **对话后一键入库**：每轮采访对话结束，自动抓取未明确、模糊的专有名词，弹窗引导用户补充完整信息，一键同步至词库，无需手动切换页面，操作极简

    - 精准匹配优化：名词库信息越完善，AI越能精准识别不同称谓指向同一人，避免记忆素材出现人物混淆，提升素材连贯性与准确性

    - 素材联动标注：梳理素材、标记图片时，可直接调用名词库内容，快速完成信息标注，减少重复输入

### 4.5 GitHub 素材存储与同步

- **仓库目录结构**：
        `/
├── articles/       # 记忆素材MD文件（按分组建目录）
├── images/         # 所有图片素材资源
├── glossary.json   # 专有名词库（含人名曾用名、别名、称谓字段，实时同步对话确认信息）
└── config.json     # 系统配置（提示词、AI设置、名词识别规则等）`

- **版本控制**：利用Git的Commit历史，完整记录每次素材修改、上传、名词确认入库操作，支持历史素材与词库回溯，保障素材可追溯。

- **跨设备访问**：通过GitHub账号登录，任意设备浏览器均可访问、编辑素材，同步更新专有名词库，打破场景限制。

### 4.6 对话后名词确认与一键入库（新增核心功能）

- **功能定位**：专属长辈的轻量化信息校准工具，不打断回忆流程，解决口述中称谓模糊、地名不详、人名混淆问题，实现专有名词实时校准、一键复用。

- **触发机制**：每轮AI对话结束（用户结束一段口述、AI完成回应后）自动触发，不强制弹窗，可选择“稍后确认”，不影响回忆节奏。

- **识别范围**：未明确全名的人名、仅有俗称/别名的称谓、表述模糊的地名、旧称/别称、特殊物品名、亲属关系模糊表述等。

- **操作流程**：
        

    1. AI自动提取本轮对话中模糊、不明确的专有名词，生成简易清单，标注“需确认”。

    2. 弹出极简弹窗，展示待确认名词及原始对话语境，用户仅需补充完整信息（如全名、准确地名、曾用名）。

    3. 点击**一键加入词库**，信息自动归类至对应名词库分类，同步更新全局词库。

    4. 选择“暂不入库”，仅记录当前素材备注，后续可在右侧名词库模块手动补充；选择“忽略”，标记为无需确认，后续不再提醒。

- **复用逻辑**：已入库名词，后续对话、素材梳理、图片标记中自动匹配，无需重复确认，支持一键调用填充，大幅简化操作。

## 5. 数据格式规范

### 5.1 记忆素材MD文件格式

```markdown
---
title: 夏夜的星空与恐惧
original_title: 夏夜乘凉
time: 幼儿园时期
location: 安徽砀山（已校准名词库）
tags: [童年, 邻里, 夏夜]
summary: 回忆儿时夏夜邻里乘凉，因听鬼故事不敢去公厕的往事。
related_images: [夏夜乘凉.jpg,老家院子.jpg]  # 关联图片素材文件名
confirmed_glossary: [王刚（标准名，曾用名王建军）、李氏（奶奶）、砀山老家院子]  # 本轮对话确认入库名词
---

# 夏夜的星空与恐惧

（素材梳理区初步整理内容，内置校准后专有名词，仅为后期写作底稿...）
```

### 5.2 图片素材元数据格式

```markdown
---
image:
  filename: 夏夜乘凉.jpg
  description: 儿时在院子里乘凉，满天繁星
  time: 1990年代
  location: 安徽砀山老家院子（词库关联）
  people: 奶奶（李氏）、邻居王刚（曾用名：王建军，称谓：王科长，词库已入库）
  url: https://raw.githubusercontent.com/xxx/images/夏夜乘凉.jpg
  tags: [童年, 星空, 老家]
  related_article: 夏夜的星空与恐惧  # 关联记忆素材标题
---
```

## 6. 非功能需求

1. **稳定性**：实时自动保存机制，确保断电/断网场景下素材、已确认名词信息不丢失，保障素材采集连续性。

2. **易用性**：界面简洁，按钮标识清晰，操作直观，名词确认弹窗极简，一键入库无复杂步骤，无学习成本，适配长辈使用习惯。

3. **安全性**：通过GitHub OAuth授权登录，用户所有素材、专有名词库仅存于自有仓库，平台无任何数据留存，保障隐私安全。

4. **性能**：MD素材解析高效，图片懒加载，名词识别、弹窗响应快速，界面无卡顿延迟，提升使用体验。

## 7. 开发里程碑

1. **Phase 1**：基础框架搭建 + GitHub素材读写功能 + 三栏布局实现。

2. **Phase 2**：AI三档交互模式开发 + 采访区实时保存功能 + 模糊名词基础识别功能。

3. **Phase 3**：素材梳理区开发 + 提示词配置 + 指令微调功能 + 对话后名词确认弹窗开发。

4. **Phase 4**：图片素材上传、存储、标记功能开发 + 专有名词库一键入库功能开发。

5. **Phase 5**：专有名词库优化 + 智能素材元数据生成 + 名词库全局联动 + 全文功能联调。
> （注：文档部分内容可能由 AI 生成）
