## 1. 产品概述

### 1.1 产品定位

**回忆录客户端（老人端）**是一款基于 Web 的单页应用（SPA），专为长辈打造轻量化、低门槛的记忆素材采集工具，核心定位为**回忆录前期素材收集终端**，仅负责通过AI辅助对话挖掘记忆碎片、留存原始口述内容，同步完成图片素材的存储与标记，所有素材统一归档留存，用于后期整合写作与AI辅助生成完整图文回忆录，不承担最终文稿编辑、排版、插图等深加工功能，最终构建结构化、可追溯、可长期保存的个人数字记忆素材库。

### 1.2 目标用户

| 属性 | 描述 |
|------|------|
| 年龄 | 70岁左右 |
| 操作能力 | 有比较强的打字和口述能力 |
| 使用方式 | 独立使用，无需子女辅助 |
| GitHub认知 | 无需了解，技术授权由系统内置 |

### 1.3 核心价值

- **零门槛素材采集**：摒弃纯文本输入模式，采用对话式交互，降低长辈创作门槛，轻松留存零散记忆。

- **AI辅助记忆挖掘**：三档可调AI参与度，从被动记录到主动引导，深度挖掘被遗忘的记忆细节，丰富素材维度。

- **永久安全素材存储**：所有文本素材、图片素材、配置文件均存储于自有GitHub仓库，依托Git实现版本控制与异地备份，素材永不丢失。

- **个性化素材整理**：支持自定义AI文风提示词，初步梳理口述素材，保留原始情感与细节，适配后期写作需求。

- **精准记忆标注**：构建专属人名/地名名词库，人名支持录入曾用名、别名、常用称谓，让AI精准识别不同称谓指向同一主体，避免记忆素材出现人物混淆，保障素材真实性。

- **实时名词校准**：每轮采访对话结束后，自动识别模糊专有名词，一键确认补充信息并同步入库，杜绝记忆信息偏差，提升素材精准度。

### 1.4 技术栈

- **前端**：HTML5 + CSS3 + JavaScript（Vue/React 任选）

- **存储**：GitHub Repository（纯文件存储，无数据库）

- **AI 接口**：对接大模型 API（用于记忆引导、对话回应、素材初步梳理、语义理解、模糊名词识别）

- **文件格式**：Markdown (MD) + JSON

### 1.5 用户角色

本产品为**双用户协作模式**，两位用户功能权限**不做任何区分**，均可在同一素材库中进行所有操作。

| 角色 | 描述 | 权限 |
|------|------|------|
| 用户A | 老人（主要口述者） | 全部功能 |
| 用户B | 子女（协助者） | 全部功能 |

## 2. 整体架构

### 2.1 架构图（文字描述）

```
用户浏览器 <--> Web 前端应用 <--> GitHub API (素材读写与备份)
                                      ^
                                      |
                                      v
                              大模型 API (记忆引导、素材梳理、模糊名词识别)
                                      ^
                                      |
                                      v
                              专有名词库 (实时同步、一键入库、智能匹配)
```

### 2.2 数据流向

1. **用户操作**：在前端进行AI对话记录、素材编辑、图片上传与标记。

2. **实时保存**：用户输入/修改后，自动提交至GitHub，无需手动点击保存，杜绝素材丢失。

3. **AI 调用**：前端将采访区原始对话内容发送至大模型，获取记忆引导、素材初步梳理结果，同步识别模糊专有名词。

4. **名词确认与入库**：用户确认模糊名词信息，一键同步至专有名词库，更新词库数据。

5. **素材读取**：前端通过GitHub API拉取仓库素材，渲染素材列表与原始内容，供后续查看与导出。

## 4. 核心功能模块

### 4.1 AI 交互模式（三档可调）

#### 4.1.1 档位定义与功能边界

| 档位 | 名称 | 适用场景 | AI响应策略 |
|------|------|----------|------------|
| **L1** | 记录档 | 长辈口述不想被打断 | 仅回复"收到"/"嗯"，不做任何追问 |
| **L2** | 倾听档 | 情感陪伴、情绪安抚 | 简短共情回应（10-20字），不提问题 |
| **L3** | 互动档 | 深度挖掘记忆细节 | 主动引导+生成3个追问问题 |

#### 4.1.2 各档位详细参数

```
L1 记录档:
├── 回复字数: 5字以内
├── 回复内容: "收到"/"嗯"/"好的，请继续"
├── 追问行为: 无
├── 模糊名词处理: 仅标记，不弹窗
└── 响应延迟: < 500ms

L2 倾听档:
├── 回复字数: 10-30字
├── 回复内容: "听起来很温馨"/"那确实很难忘"/共情语
├── 追问行为: 无
├── 模糊名词处理: 标记+弹窗确认
└── 响应延迟: < 800ms

L3 互动档:
├── 回复字数: 50-150字
├── 回复内容: 深度回应 + 3个引导性问题
├── 追问行为: 用户可点击问题继续深挖
├── 模糊名词处理: 标记+弹窗确认+一键入库
├── 引导问题生成: 基于本轮对话内容生成
└── 响应延迟: < 1500ms
```

#### 4.1.3 档位切换交互设计

```
位置: 中间栏顶部工具栏

交互方式:
1. 快捷切换按钮组：[记录] [倾听] [互动]
2. 鼠标悬停显示档位说明 tooltip
3. 切换时显示当前档位标签 200ms 高亮动画
```

### 4.2 智能素材生成

- **新建素材流程**：

    1. 用户点击"新建"，输入**素材简要标题**（如：夏夜乘凉），标注记忆主题。

    2. 进入采访区，与AI开启对话，记录原始口述记忆。

    3. 单轮对话结束，AI自动识别模糊人名、地名、称谓、旧称等信息，弹出**名词确认弹窗**。

    4. 用户补充完整信息，点击**一键加入词库**，信息自动同步至专有名词库，无需手动跳转编辑。

    5. 内容记录完成后，点击"梳理素材"。

    6. AI自动分析原始对话内容，结合已校准名词库，生成基础素材信息：

        - **优化素材标题**

        - **时间/时段标签**（如：幼儿园时期）

        - **素材核心摘要**

        - **初步梳理文稿**（内置校准后专有名词，无信息偏差）

    7. 以上元数据与梳理文稿自动置于素材梳理区头部，完整归档单条记忆素材。

### 4.3 专有名词库（Glossary）

#### 4.3.1 词库复用触发机制

```typescript
interface NounTrigger {
  type: "person" | "location" | "item" | "relation";
  rawText: string;        // 原始模糊表述
  confidence: number;     // 置信度 0-1
  context: string;        // 上下文句子
  position: { line: number; start: number; end: number };
}

const TRIGGER_RULES = {
  person: {
    minConfidence: 0.6,
    patterns: ["/^.{1,3}$/", "/^小.+$/", "/^老.+$/", "/表.+/"],
    exclude: ["爸爸", "妈妈", "爷爷", "奶奶", "我", "你"]
  },
  location: {
    minConfidence: 0.5,
    patterns: ["/那(儿|里)/", "/这(儿|里)/", "/老家/"],
    needConfirmation: true
  },
  item: {
    minConfidence: 0.7,
    patterns: ["/旧.+/", "/以前.+/"],
    needConfirmation: true
  }
};
```

#### 4.3.2 弹窗确认流程

```

├─────────────────────────────────────────────────────────┤
│  发现以下需要确认的信息：                                │
│                                                         │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 👤 "王科长"                                     │    │
│  │ 原文：那时候王科长经常来我们家玩                │    │

```

### 5.2 异常处理流程

```
异常场景 A: 网络断开
├── 检测: 发送前 ping 检测
├── 响应: 显示"网络不稳定，内容已缓存本地"
├── 恢复: 网络恢复后自动重试
└── 数据: localStorage 暂存 + 队列管理

异常场景 B: AI 响应超时 ( > 5s )
├── 响应: 显示"AI思考中，请稍候..."
├── 重试: 3次自动重试
├── 放弃: 显示"AI暂时无法响应，是否保存原始内容？"
└── 回退: 用户可选择保存原始对话

异常场景 C: 词库冲突
├── 场景: 检测到相似词条
├── 响应: 弹窗"检测到相似词条，请确认是否合并"
└── 选项: [合并] [保留两者] [取消]

异常场景 D: 双用户同时编辑冲突
├── 场景: 两位用户同时编辑同一素材
├── 响应: 无提示，后提交者覆盖
└── 追溯: Git 版本历史可查看每次修改
```
